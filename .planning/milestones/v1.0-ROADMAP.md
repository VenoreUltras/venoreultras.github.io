## Overview

Brownfield extension symulatora PM-300 — z prostego demo kinematyki slider-crank do pełnego trenażera SOP (Standard Operating Procedure). Sześć faz dependency-correct: pure logika SOP (P1) → 13 interaktywnych meshy (P2) → click-to-state pipeline (P3) → visual feedback layer z redundant encoding (P4) → warstwa edukacyjna z tooltipami/audio/3D labels (P5) → 4 scenariusze + replay + retry + JSON/PDF eksport (P6).

**Definition of Done v1.0:** Operator w przeglądarce uczy się wykonując 4 scenariusze SOP (uruchomienie / cykl-pracy / zatrzymanie / awaria), klika rzeczywiste komponenty 3D, otrzymuje natychmiastowy visual+audio feedback w trybie Nauka (retry button) lub czysto-egzaminacyjny scoring w trybie Egzamin, a sesja jest odtwarzalna (replay z scrubberem) i eksportowalna do polskiego PDF/JSON.

## Milestone Summary

**Key Decisions:**
- Brownfield extension over rewrite — zachowanie istniejącego PressModel/SceneSetup/PhysicsEngine/UI
- Polski-only język UI + i18n centralized w `src/i18n/pl.js`
- Zustand vanilla + `subscribeWithSelector` zamiast obserwerów/event bus
- GSAP ticker (NIE requestAnimationFrame) jako single source of truth dla timingu — pinned `~3.15.0`
- `tests/boundaries.test.js` jako structural enforcement zamiast docs
- Deklaratywne JSON scenariusze + pure ProcedureEngine — testowalne w Node bez DOM
- Per-mesh cloned `MeshStandardMaterial` — eliminacja CRIT-6 (shared material highlight leak)
- Redundant encoding (color + icon + text) — accessibility WCAG enforced od Phase 1
