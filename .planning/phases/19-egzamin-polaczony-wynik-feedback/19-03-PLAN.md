<task type="auto">
  <name>Task 1: Reguły CSS sizing/scroll modala bhp-quiz</name>
  <read_first>
    - style.css (linie 685-712 — .modal-card i warianty; linia 934 — .modal-card__body; blok 1615+ .bhp-quiz__*)
    - src/ui/QuizController.js (linie 53-114 — struktura dialog/header/body/actions, klasy)
  </read_first>
  <action>
    W style.css dodaj wariant `.modal-card--bhp-quiz`:
    - `display: flex; flex-direction: column;` aby header / body / actions układały się pionowo.
    - `max-height: min(80vh, 660px);` (lub równoważnie zapewniając mieszczenie na 720px wys. z marginesem) — Claude's Discretion co do dokładnej wartości, byle przy 720px wysokości viewportu modal nie był ucięty.
    - `overflow: hidden;` (scroll przenosimy do body, nie na całą kartę) — nadpisuje dziedziczony `overflow-y:auto` z `.modal-card`.
    Dodaj regułę scrolla body w obrębie tego wariantu: `.modal-card--bhp-quiz .modal-card__body { flex: 1 1 auto; min-height: 0; overflow-y: auto; overflow-x: hidden; }`.
    Zapobiegnij poziomemu scrollowi/przepełnieniu długich słów: w `.bhp-quiz__question` (lub `.bhp-quiz`) dodaj `overflow-wrap: anywhere;` (lub `word-break`), bez psucia istniejącego layoutu.
    Upewnij się, że `.modal-card__header` i `.modal-card__actions` w tym wariancie pozostają `flex: 0 0 auto` (nie kurczą się/nie znikają przy długim body).
    Nie zmieniaj zachowania innych modali (zmiany zakresowane do `.modal-card--bhp-quiz`). Komentarz po polsku opisujący cel (QUIZ-02).
  </action>
  <acceptance_criteria>

    - `grep -nE "modal-card--bhp-quiz .modal-card__body" style.css` pokazuje `overflow-y: auto` i `min-height: 0`.
    - `grep -nE "overflow-wrap|word-break" style.css | grep -iE "bhp-quiz"` pokazuje zabezpieczenie przed poziomym przepełnieniem.
    - `npm run build` przechodzi.
    - `npm test` bez regresji (CSS nie łamie testów jsdom).
  </acceptance_criteria>
