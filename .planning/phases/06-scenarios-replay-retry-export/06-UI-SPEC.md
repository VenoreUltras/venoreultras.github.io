### 2. Replay Drawer (.replay-drawer)

**Wyzwalacz:** (a) sesja zakonczona (`state.session.finishedAt !== null`) — drawer automatycznie
wysuwany; (b) klik "Otwórz replay" w session-overlay; (c) klik "Wyswietl ostatnia sesje"
w StatusPanel (gdy persisted session dostepna, D-Phase6-12).
**Zamkniecie:** klik "Zamknij replay" button ALBO klik X w naroznik drawera.
**Pozycja:** `position: fixed; bottom: 0; left: 0; right: 0; height: 140px; z-index: 200`
(nad sceną 3D z-index:1, nad #ui-layer z-index:10, pod #modal-container z-index:300).

Struktura DOM:
```html
<div class="replay-drawer" id="replay-drawer" aria-label="Replay sesji" role="region">
  <div class="replay-drawer__toolbar">
    <button class="replay-drawer__play-pause" aria-label="Odtwarzaj / Pauza">▶</button>
    <input type="range" class="replay-drawer__scrubber" min="0" step="1"
           aria-label="Pozycja w sesji" />
    <span class="replay-drawer__timestamp" aria-live="polite"></span>
    <button class="replay-drawer__speed" aria-label="Szybkość odtwarzania">1×</button>
    <button class="replay-drawer__close" aria-label="Zamknij replay">✕</button>
  </div>
  <div class="replay-drawer__info">
    <!-- Nazwa scenariusza + attempt index -->
  </div>
</div>
```

Wyglad:
- `height: 140px` (locked D-Phase6-07)
- Background: `rgba(16, 24, 39, 0.95)` + `backdrop-filter: blur(12px)`
- Border-top: `1px solid rgba(255, 255, 255, 0.12)`
- Box-shadow: `0 -8px 24px rgba(0, 0, 0, 0.4)` (shadow upward)
- Padding: 12px 16px (sm x md)
- Wejscie: `transform: translateY(0)` — animacja wysuwania `transform translateY(140px) → 0`
  przez 250ms `ease-out`
- Wyjscie: `transform: translateY(140px)` przez 200ms `ease-in`

`.replay-drawer__toolbar`:
- `display: flex; align-items: center; gap: 12px`
- Wysokosc: ~48px (button + scrubber)

`.replay-drawer__scrubber`:
- `flex: 1`
- Height thumb: 16px, height track: 4px
- Track background: `rgba(255,255,255,0.15)` fill / `var(--accent-primary)` progress
- Thumb: 16px kolo, `background: var(--accent-primary)`, `box-shadow: 0 0 8px rgba(59,130,246,0.5)`
- Focus-visible: outline 2px accent, outline-offset 2px

`.replay-drawer__timestamp`:
- Font: JetBrains Mono, 13px, `--text-muted`
- Format: `MM:SS / MM:SS` (current / total)
- `aria-live: polite` — ekran czytnikowy nie musi byc notified per scrub

`.replay-drawer__play-pause`:
- `min-width: 44px; min-height: 44px`
- Background: transparent; border: `1px solid var(--glass-border)`; border-radius 8px
- Tekst: "▶" (idle) / "⏸" (playing)
- Active (playing): `background: rgba(59,130,246,0.15); border-color: var(--accent-primary)`
- Hover: `background: rgba(255,255,255,0.06)`

`.replay-drawer__speed`:
- `min-width: 44px; min-height: 44px`
- Tekst: "1×" (normal) / "0.25×" (slow-mo) — toggle
- Slow-mo aktywny: `color: #F0E442; border-color: rgba(240,228,66,0.5)` (Wong yellow)
- Font: 13px semibold

`.replay-drawer__close`:
- `min-width: 44px; min-height: 44px` (WCAG touch target)
- Background: transparent; border: `1px solid var(--glass-border)`; border-radius 6px
- Kolor: `var(--text-muted)` → hover `var(--text-main)`

`.replay-drawer__info`:
- Font: 13px / `--text-muted`
- Tresc: `{scenarioTitle} · Próba {attemptIdx + 1} z {total}`
- Padding-top: 4px (xs)

Maszyna stanow drawera:
```
zamkniety → triggerReplay() → wysuniecie (250ms) → otwarty
otwarty + playing → auto-scrub → done → pauza na ostatnim evencie
otwarty → close button → wyjscie (200ms) → zamkniety
```

### 4. Retry Button (.step-item__retry)

**Warunek:** `state.difficulty === 'nauka'` AND aktywny krok ma status `'error'`.
**Lokalizacja:** w StepPanel, inline pod aktywnym error step (po `.step-item__rationale`
  lub bezposrednio pod label gdy rationale brak).
**Brak w Egzamin:** element DOM nie jest tworzony (branch warunkowy, nie `display:none`).

Wyglad `.step-item__retry`:
- `display: inline-flex; align-items: center; gap: 4px`
- Padding: 6px 12px (miedzy sm i md)
- Border-radius: 6px
- Font: 13px / weight 600
- Background: `rgba(240, 228, 66, 0.12)` (podtonowany Wong yellow)
- Border: `1px solid rgba(240, 228, 66, 0.4)`
- Kolor tekstu: `#F0E442`
- `margin-top: 6px` (nie z 8-point grid — pragmatyczny odstep od rationale)
- Hover: `background: rgba(240, 228, 66, 0.20); border-color: rgba(240, 228, 66, 0.6)`
- Focus-visible: `outline: 2px solid #F0E442; outline-offset: 2px`
- Tresc: `pl.overlay.retry` — "Spróbuj ponownie" (bez emoji — button, nie badge)

Maszyna stanow:
```
krok.status = 'error' + nauka → przycisk widoczny

krok.status = inne → przycisk nie renderuje sie
```

### 5. Bimanual Feedback (.bimanual-hint)

**Dotyczy:** scenariusz cykl-pracy, krok 5 (`kind: 'bimanual'`, D-Phase6-04).
**Wyzwalacz:** `store.attemptBimanualStep` po pierwszym kliku (lewy LUB prawy przycisk start).
**Cel:** pokazac kursantowi ze okno 500ms jest aktywne i ile czasu zostalo.

Warianty implementacji (planner wybiera jedna z dwoch):
- **Wariant A (rekomendowany):** progress bar w `.step-item--aktywny` — waska linia
  pod tekstem kroku, animacja od 100% do 0% przez 500ms.
- **Wariant B:** popup tooltip na meshu pierwszego przycisku.

Kontrakt dla Wariantu A (`.bimanual-hint`):
- `height: 4px; border-radius: 2px`
- `margin-top: 4px` (xs)
- Background track: `rgba(255,255,255,0.1)`
- Fill: `rgba(240, 228, 66, 0.8)` (Wong yellow) animowany width 100%→0% przez 500ms linear
- Po timeout (>500ms od pierwszego kliku bez drugiego):
  fill zmienia sie na `rgba(213, 94, 0, 0.8)` (Wong vermillion) + flash 200ms → znika
- Po sukcesie (drugi klik w czasie):
  fill zmienia sie na `rgba(0, 158, 115, 0.8)` (Wong green) + znika w 300ms
- CSS animacja: `@keyframes bimanual-timer { from {width: 100%} to {width: 0%} }`
  z `animation-duration: 500ms; animation-timing-function: linear; animation-fill-mode: forwards`

CSS klasy:
```
.bimanual-hint                  — kontener progress bar (normalnie ukryty)
.bimanual-hint--active          — widoczny, animacja aktywna
.bimanual-hint--timeout         — fill czerwony (error)
.bimanual-hint--success         — fill zielony (success, zanika)
```

## Interakcje i stany komponentow

### Maszyna stanow session-overlay

```
null → session.finishedAt ustawione → otwarcie auto (250ms fade-in)
otwarty → klik X | klik backdrop → zamkniecie (150ms fade-out) → null
otwarty → klik "Otwórz replay" → otwarcie replay-drawer + overlay pozostaje
otwarty + Nauka → klik "Spróbuj ponownie" → zamkniecie overlay + store.retry()
```

### Maszyna stanow replay-drawer

```
zamkniety → triggerReplay() | klik "Otwórz replay" → wysuniecie (250ms)
otwarty idle → klik play → granie (auto-scrub)
otwarty granie → klik pauza | scrub manualny → pauza
otwarty → speed toggle → zmiana tickRate (1x / 0.25x), nie restart
otwarty → klik zamknij → wyjscie (200ms)
koniec events → auto-pauza na ostatnim evencie
```

### Maszyna stanow bimanual-hint

```
krok inactive → hint ukryty
krok aktywny + kind='bimanual' → hint pojawia sie (nie animuje jeszcze)
pierwszy klik przycisku → .bimanual-hint--active → animacja 500ms start
drugi klik w czasie → .bimanual-hint--success → fill zielony → zanika 300ms
brak kliku po 500ms → .bimanual-hint--timeout → fill czerwony → flash + zanika
```

### Scenariusz selector — zachowanie scroll vs klik

```
1 scenariusz aktywny, klik inny, sesja nieaktywna → natychmiastowy load
1 scenariusz aktywny, klik inny, mid-run → ConfirmModal (Phase 5 reuse)
1 scenariusz aktywny, klik ten sam → no-op
keyboard 1-4 → identyczne jak klik odpowiedniego scenario-btn
```
