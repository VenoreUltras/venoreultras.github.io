Algorithm: iteruj po events, śledź `expectedNextId` z scenario.steps; gdy step.done.stepId !== expectedNextId, push {from: expectedNextId, to: event.stepId}. Jeśli scenario undefined, sequenceViolations=[].

Dla overall metric (across attempts), Plan 06-07 wywoła computeMetrics per-attempt i agreguje. Dla tego planu: `retryCount = 0` (per-attempt level; agregat w Plan 06-07).
    - `score`: deleguj do `calculate(events, opts)`.

    ```
    {
      errorCount, criticalCount, mediumCount, minorCount,
      completionTimeMs, missedSteps: string[],
      sequenceViolations: Array<{from, to}>,
      retryCount, score
    }
    ```

    - computeMetrics bez scenario param → missedSteps i sequenceViolations to []; reszta liczona normalnie.

  <files>src/persistence/sessionPersistence.js, tests/sessionPersistence.test.js, tests/boundaries.test.js</files>
  <behavior>
    Nowy moduł `src/persistence/sessionPersistence.js`. Eksporty:

    ```
    export const SESSION_KEY = 'pm300:session:v1';

    export function loadPersistedSession(key = SESSION_KEY): SessionSnapshot | null
    export function savePersistedSession(snapshot, key = SESSION_KEY): boolean  // true on success, false on quota/private
    export function clearPersistedSession(key = SESSION_KEY): void
    ```

    `loadPersistedSession`:
    - Try: `const raw = localStorage.getItem(key)`. Jeśli null → return null.
    - Try: `const obj = JSON.parse(raw)`. Catch SyntaxError → console.warn polski + clearPersistedSession + return null.
    - Sprawdź `obj.version === 'v1'` AND `obj.session && typeof obj.session === 'object'` AND `Array.isArray(obj.session.attempts)` AND `typeof obj.session.scenarioId === 'string'`. Jeśli brak któregoś → console.warn + clearPersistedSession + return null.
    - Walidacja głębsza: każdy attempt ma `events: Array`, `attemptIdx: number`. Wzajemne odrzucenie tylko jeśli zafield obowiązkowy źle ukształtowany.
    - Return validated obj.

    `savePersistedSession(snapshot)`:
    - Try: `localStorage.setItem(key, JSON.stringify(snapshot))`. Catch (QuotaExceededError, private mode) → console.warn polski + return false.
    - Return true.

    Schema snapshot:
    ```
    {
      version: 'v1',
      session: { scenarioId, startedAt, finishedAt, attempts:[...], retryCount },
      metadata: { exportedAt: number, appVersion: 'pm300-trener v1.0', scenarioTitlePL?: string }
    }
    ```

    `clearPersistedSession`: `try { localStorage.removeItem(key); } catch {}`.

    Testy (`tests/sessionPersistence.test.js`):
    - localStorage empty → loadPersistedSession returns null.
    - localStorage = 'not-json{' → loadPersistedSession returns null + clear.
    - localStorage = '{"version":"v0",...}' → returns null + clear.
    - localStorage = '{"version":"v1","session":{...valid...}}' → returns object.
    - localStorage = '{"version":"v1","session":{scenarioId:"x"}}' (brak attempts) → returns null.
    - savePersistedSession({version:'v1',session:{...}}) → localStorage zawiera serialized JSON; return true.
    - savePersistedSession z mock localStorage.setItem rzucającym QuotaExceededError → return false; console.warn called.
    - clearPersistedSession usuwa key.

    ~8 asercji. Użyj `vi.spyOn(global.localStorage, 'getItem')` lub `Object.defineProperty(global, 'localStorage', {...mock})`.

    Boundary entry: `src/persistence/sessionPersistence.js` → `{ mustNotImport: ['three', 'gsap', '../training/', '../state/', '../ui/', '../highlight/', '../education/', '@floating-ui/dom'] }`. Pure utility module dotyczący tylko localStorage + JSON.
