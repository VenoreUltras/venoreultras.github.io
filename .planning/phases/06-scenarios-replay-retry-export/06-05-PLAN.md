  <name>Task 2: RaycastController — bimanual pointerdown flow + integration z store</name>
  <files>src/RaycastController.js, tests/RaycastController.test.js</files>
  <behavior>
    RaycastController._handlePointerDown (lub analogiczna metoda — patrz istniejący kod) rozszerzona o bimanual:

    Pole instance: `_lastBimanualDown = null` ({ meshId, timestamp }).
    Pole instance: `_bimanualTimeoutHandle = null` (handle setTimeout).

    Logika:
    - Po raycaście, jeśli mesh klikany jest hit AND `state.currentStep.kind === 'bimanual'` AND meshId należy do `state.activeScenario.steps[currentStepId].targetMeshIds`:
      - Jeśli `_lastBimanualDown === null`:
        - `_lastBimanualDown = { meshId, timestamp: Date.now() }`.
        - `store.setBimanualHintState('active')`.
        - `clearTimeout(_bimanualTimeoutHandle)`.
        - `_bimanualTimeoutHandle = setTimeout(() => { store.setBimanualHintState('timeout'); _lastBimanualDown = null; setTimeout(() => store.setBimanualHintState('idle'), 600); }, windowMs)` (windowMs z step.windowMs ?? 500).
        - Return.
      - Jeśli `_lastBimanualDown !== null`:
        - Sprawdź czy nowy meshId !== _lastBimanualDown.meshId (oba różne) i oba w targetMeshIds.
        - Dispatch: `store.attemptBimanualStep({firstMeshId: _lastBimanualDown.meshId, firstTimestamp: _lastBimanualDown.timestamp, secondMeshId: meshId, secondTimestamp: Date.now()})`.
        - `clearTimeout(_bimanualTimeoutHandle)`.
        - `_lastBimanualDown = null`.
        - Jeśli wynik attemptBimanualStep advansował step (steps[stepId].status==='done' po dispatch) → `store.setBimanualHintState('success')`, setTimeout(() => 'idle', 300).

    NIE delegujesz do `_handlePointerUp` (istniejący Phase 3-5 path) gdy bimanual aktywny — pierwszy klik na bimanual mesh NIE wywołuje `attemptStep` zwykłego.

    Brownfield edit: zachowaj istniejący kontrakt dla manipulation/visual-target steps. Tylko gdy `currentStep.kind === 'bimanual'` AND `mesh.userData.id in step.targetMeshIds` wchodzimy w bimanual branch.

    Helper method: `_getCurrentStep()` — `const s = this._store.getState(); return s.activeScenario?.steps?.find(x => x.id === s.currentStepId);`.

    Dispose extension: `clearTimeout(_bimanualTimeoutHandle)`.

    Testy (tests/RaycastController.test.js — dodaj `describe('Phase 6 — bimanual')`):
    - Setup: scenariusz z 1 stepem bimanual targetMeshIds=['left','right'], windowMs=500. Mock interactables z 2 meshami.
    - Pierwszy click 'left' → store.setBimanualHintState('active') called; store.attemptBimanualStep NIE called yet; _lastBimanualDown != null.
    - Drugi click 'right' w 100ms → store.attemptBimanualStep called z {firstMeshId:'left', secondMeshId:'right', firstTimestamp, secondTimestamp}.
    - Pierwszy click 'left' + brak drugiego po 500ms (vi.advanceTimersByTime(500)) → store.setBimanualHintState('timeout') called.
    - Pierwszy click 'left' + drugi click ten sam 'left' → no-op (nie dispatch attemptBimanualStep, hint pozostaje active do timeout).
    - Klik na mesh NIE w targetMeshIds → fall-through do zwykłego attemptStep flow (nie bimanual).
    - Kind manipulation step → bimanual branch nie odpala, istniejący test attemptStep wciąż zielony.
    - dispose() → clearTimeout dla bimanual handle.

    ~8 nowych asercji.
