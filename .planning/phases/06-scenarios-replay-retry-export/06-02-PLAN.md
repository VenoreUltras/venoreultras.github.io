    ```
    store.subscribe(
      (s) => s.currentStepId,
      (currentStepId, prevStepId) => {
        const s = store.getState();
        // Gdy currentStepId staje się null (advanceStep przeszedł poza ostatni krok) — finishSession

        }
      }
    );
    ```

Pure logic ProcedureEngine już to zwraca jako `{ok:false, reason:'machine-state-not-matching', effects:[]}` z Plan 06-01 Task 2.

    3. **Bimanual lock**: gdy `attemptBimanualStep` jest in-flight (isAnimating=true), drugi attemptBimanualStep zwraca no-op (analog attemptStep CRIT-8).

    4. **Retry w trakcie attemptStep** (rzadkie, ale możliwe): `retry()` jest synchroniczne i mutuje state przez `set`. NIE wymaga isAnimating guarda (retry to user-initiated reset, nie sub-step walidacja).

    - machineStateAttest advansuje gdy machineState === target (jednorazowo, nie wpada w pętlę dzięki advanceStep idempotency lock z Phase 3).
