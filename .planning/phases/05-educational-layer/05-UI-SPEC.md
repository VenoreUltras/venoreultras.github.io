# Phase 5 — UI Design Contract

> Visual and interaction contract for Phase 5: Educational Layer.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.
>
> Faza 5 dodaje 5 nowych powierzchni UI na wierzch istniejacego systemu glassmorphism (Phase 1-4).
> Kontrakt rozszerza, nie zastepuje, zmienne CSS zdefiniowane w style.css (Phase 4).

## Design System

| Property    | Value                                                                              |
|-------------|------------------------------------------------------------------------------------|
| Tool        | none — vanilla HTML/CSS/JS; brak shadcn/Radix/React                               |
| Preset      | not applicable                                                                     |
| Component library | none (DOM direct, klasy JS per CONVENTIONS.md)                             |
| Icon library | emoji (glify Unicode; OS-rendering zaakceptowany swiadomie — D-Phase4-05)         |

## Powierzchnie UI Phase 5 — Szczegolowy Kontrakt

### 1. Tooltip (.tooltip)

**Wyzwalacz:** hover na interactable mesh (RaycastController callback), 600ms delay (D-Phase5-08).
**Stan active:** tylko gdy `difficulty === 'nauka'` AND `activeModal === null`.
**Pozycjonowanie:** `@floating-ui/dom` — `placement: 'top'`, `flip()` + `shift({ padding: 8px })`.
**Aktualizacja:** `autoUpdate(referenceEl, tooltipEl, compute)` — reaguje na scroll + resize.
**Implementacja DOM:** single element `<div class="tooltip" role="tooltip">` appended do `document.body`; content swap (nie re-create per hover).

Wymiary i wyglad:
- `max-width: 240px`
- Padding: 8px 12px (`sm` x `md/1.5`)
- Border-radius: 8px (zgodne z `.btn` i `.step-item`)
- Font: 14px / weight 400 / line-height 1.5
- Tlo: `rgba(16, 24, 39, 0.92)` + `backdrop-filter: blur(8px)`
- Border: `1px solid rgba(255,255,255,0.15)`
- Box-shadow: `0 4px 12px rgba(0,0,0,0.4)`
- Kolor tekstu: `var(--text-main)` (#f3f4f6)
- `position: absolute` (floating-ui wymaga)
- `pointer-events: none` (nie blokuje hover na meshu)
- Animacja wejscia: `opacity 0 → 1` w 120ms `ease-out` (subtelne, nie rozpraszajace)
- Animacja wyjscia: natychmiastowe `display: none` (bez opoznienia przy leave)

Tresc: `pl.parts[meshId].description` (jednozdaniowy opis dydaktyczny, 80-160 znakow, PL).
Zawartosc wyswietlana jako `textContent` (XSS-safe).

CSS klasy:
```
.tooltip                  — element bazowy (zawsze w DOM po build)
.tooltip--visible         — stan widoczny (opacity:1, display:block)
.tooltip--hidden          — stan ukryty (opacity:0, display:none)
```

### 2. Etykiety 3D (.label-3d, LabelOverlay)

**Wyzwalacz:** klawisz `L` toggle `state.labelsVisible` — aktywny tylko gdy `difficulty !== 'egzamin'`.
**Renderer:** `CSS2DRenderer` z `three/addons/renderers/CSS2DRenderer.js`.
**Mount:** `<div id="label-overlay-container">` w `index.html` jako sibling `#three-canvas` (pointer-events: none na calym kontenerze).
**Filtrowanie:** camera-facing (dot product worldNormal vs cameraDir > 0 per frame).

Tresc: `mesh.userData.labelPL` (krotki rzeczownik, np. "Koło zamachowe") — bezposrednio z userData, nie z pl.js (to jest identity field per D-Phase2-04).

### 3. HelpModal (.modal-overlay + .modal-card)

**Wyzwalacz:** klawisz `H` toggle / `store.toggleHelp()` → `state.activeModal === 'help'`.
**Zamkniecie:** `H` ponownie | `Esc` | klik `[X]` button | klik `modal-overlay` poza karta.
**Efekt na fizyke:** `activeModal !== null` pauzuje `currentAngle += omega * dt` w gsap ticker (D-Phase5-28). Rendering kontynuuje.
**Focus trap:** po otwarciu focus przechodzi na pierwszy element interaktywny w karcie (przycisk `[X]`); Tab krazsy w srodku; Shift+Tab dziala odwrotnie; focus nie wycieka poza modal.

Struktura DOM:
```
<div id="modal-container">
  <div class="modal-overlay" aria-hidden="true"></div>
  <dialog class="modal-card" role="dialog" aria-modal="true" aria-labelledby="modal-title">
    <header class="modal-card__header">
      <h2 id="modal-title" class="modal-card__title"></h2>
      <button class="modal-card__close" type="button" aria-label="Zamknij pomoc">✕</button>
    </header>
    <div class="modal-card__body">
      <!-- sekcje: keymap / legenda kolorow / legenda ikon / disclaimer -->
    </div>
  </dialog>
</div>
```

Uwaga: uzycie `<dialog>` (native HTML) zamiast `<div role="dialog">` — zapewnia focus trap i semantyke bez dodatkowego JS; fallback do `<div>` jesli jsdom/test env nie obsluguje.

Wyglad `.modal-overlay`:
- `position: fixed; inset: 0`
- `background: rgba(11, 15, 25, 0.85)`
- `z-index: 300`

Wyglad `.modal-card`:
- `position: fixed`
- `top: 50%; left: 50%; transform: translate(-50%, -50%)`
- `z-index: 301`
- `width: min(640px, 92vw)`
- `max-height: 80vh`
- `overflow-y: auto`
- Tlo: `var(--glass-bg)` + `backdrop-filter: blur(12px)`
- Border: `1px solid var(--glass-border)`
- Border-radius: 16px
- Padding: 24px (`lg`)
- Box-shadow: `0 25px 50px -12px rgba(0,0,0,0.6)`

Wyglad naglowka `.modal-card__header`:
- `display: flex; justify-content: space-between; align-items: center`
- `margin-bottom: 16px` (md)
- `padding-bottom: 12px`
- `border-bottom: 1px solid var(--glass-border)`

`.modal-card__title`:
- Font: 16px / weight 600
- Kolor: `var(--text-main)`

`.modal-card__close`:
- `min-width: 44px; min-height: 44px` (WCAG touch target)
- Background: transparent
- Border: `1px solid var(--glass-border)`
- Border-radius: 6px
- Kolor: `var(--text-muted)`
- Hover: `background: rgba(255,255,255,0.05)`
- Focus-visible: `outline: 2px solid var(--accent-primary); outline-offset: 2px`

### 4. ConfirmModal (generic Modal, wariant .modal-card--confirm)

**Wyzwalacz:** klawisz `1-4` gdy aktywna procedura (`currentStepId !== null` AND nie wszystkie kroki done) → `store.openConfirmModal(payload)` → `state.activeModal === 'confirm-scenario-switch'`.
**Zamkniecie / Anuluj:** `Esc` | klik "Anuluj" → `store.closeModal()`; powrot do procedury bez zmian.
**Potwierdzenie:** klik "Zaladuj scenariusz" → `store.loadScenario(payload.newScenario)` + `store.closeModal()`.

Struktura: identyczna z HelpModal (generic Modal base) z innym `aria-labelledby` i `modal-card__body` zawierajacym tylko tekst pytania + przyciski akcji.

Tresc:
- Tytul: `pl.modals.confirmScenarioSwitch.title` — "Zmiana scenariusza"
- Body: `pl.modals.confirmScenarioSwitch.body(nazwaAktualnego, nazwaNowego)` — "Przerwie postep w '{nazwaAktualnego}'. Zaladowac '{nazwaNowego}'?"
- Przycisk primary CTA: "Zaladuj scenariusz" (kolor `var(--accent-primary)`)
- Przycisk cancel: "Anuluj" (styl `.btn.secondary` z style.css)

Przyciski akcji `.modal-card__actions`:
- `display: flex; justify-content: flex-end; gap: 12px`
- `margin-top: 24px` (lg)
- Primary: `.btn.primary` (istniejaca klasa) — "Zaladuj scenariusz"
- Cancel: `.btn.secondary` (istniejaca klasa) — "Anuluj"

## Interakcje i stany komponentow

### Maszyna stanow tooltip

```
hidden → (hover wchodzi na mesh + delay 600ms + nauka + !modal) → visible
visible → (hover schodzi) → hidden (natychmiastowe)
visible → (activeModal !== null LUB difficulty=egzamin) → hidden (natychmiastowe)
hidden → (aktywny modal) → blokada (hover timer czyszczony)
```

### Maszyna stanow modali

```
null → H → 'help'
null → 1-4 mid-run → 'confirm-scenario-switch'
'help' → H | Esc | X | klik-poza → null
'confirm-scenario-switch' → Anuluj | Esc → null
'confirm-scenario-switch' → Zaladuj → null + loadScenario()
null → Esc → triggerEStop() (Phase 1 SOP-09 invariant)
```

### Maszyna stanow etykiet 3D

```
false → L (tylko nauka) → true (labele widoczne)
true → L → false (labele ukryte)
true + difficulty zmienione na egzamin → force-false (K no-op)
true → camera obroci sie → per-frame filter (dot product) ukrywa/pokazuje wybrane
```
