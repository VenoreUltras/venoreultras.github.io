# Phase 1 — UI Design Contract

> Visual and interaction contract for Phase 1: Foundation.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.
>
> **Scope constraint:** Phase 1 delivers exactly two UI deliverables:
> 1. Disclaimer banner component (UI-05) — sticky top, collapsible
> 2. `src/i18n/pl.js` translation table (UI-06) — single source for all Polish strings
>
> No new panels, no 3D UI, no tooltips. Those land in Phases 4 and 5.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gap inside banner (between `!` icon and text) |
| sm | 8px | Banner vertical padding in collapsed state; chevron button padding |
| md | 16px | Banner horizontal padding; gap between icon and disclaimer text in expanded state |
| lg | 24px | Not used in Phase 1 UI |
| xl | 32px | Not used in Phase 1 UI |
| 2xl | 48px | Not used in Phase 1 UI |
| 3xl | 64px | Not used in Phase 1 UI |

Exceptions:
- Touch target for the collapse/expand chevron button: minimum 44×44px (accessibility, pointer target size). This exceeds the 8-point scale but is a WCAG 2.5.5 hard floor.
- Banner height expanded: 32px (2× `md/2` = 16px padding top+bottom + ~14px text = ~32px total). Use `min-height: 32px`.
- Banner height collapsed: 8px visible bar + icon centered at 44px touch target (the bar is 8px visually but the `<button>` must be 44px tall, positioned so the bar is its bottom border).

## Color

Wong color-blind safe palette is a hard constraint for this project (CRIT-4, ROADMAP cross-cutting invariants). All semantic color choices must use Wong values.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0b0f19` | Background surface — `--bg-color` from existing `style.css` |
| Secondary (30%) | `rgba(16, 24, 39, 0.65)` | Glass panels — `--glass-bg` from existing `style.css` |
| Accent (10%) | `#3b82f6` | Reserved: interactive affordances only (existing CTA button, slider thumb) — NOT used in disclaimer banner |
| Semantic — Warning/Disclaimer | `#E69F00` | Wong amber — disclaimer banner background in expanded state (color channel of redundant encoding) |
| Semantic — Error | `#D55E00` | Wong vermillion — error states, fault indicators (Phase 4 enforces this; Phase 1 locks the token in `pl.js` comments and CSS) |
| Semantic — Success | `#009E73` | Wong teal — success/correct step states (Phase 4 enforces; Phase 1 locks token) |

**Disclaimer banner color breakdown (redundant encoding = color + icon + text):**

| State | Background | Text | Icon | Border |
|-------|-----------|------|------|--------|
| Expanded | `#E69F00` at 15% opacity over `--glass-bg` (`rgba(230,159,0,0.15)`) | `#f3f4f6` (`--text-main`) | `!` icon in `#E69F00` | `1px solid rgba(230,159,0,0.4)` |
| Collapsed (1px bar) | `#E69F00` at 60% opacity | — | `!` button visible, `#E69F00` | left border `3px solid #E69F00` |

Wong amber `#E69F00` chosen over blue accent because: (1) it signals "caution/notice" semantically, (2) it is distinct under deuteranopia and protanopia simulations, (3) it does not collide with the error color `#D55E00` or success color `#009E73`.

Accent reserved for: interactive button CTA states, range slider thumb, link hover. Never used for disclaimer / warning / error / success semantics.

## Component Inventory

### DisclaimerBanner

The only new DOM component in Phase 1.

**HTML structure:**

```
<div id="disclaimer-banner" class="disclaimer-banner" role="region" aria-label="Zastrzeżenie symulatora">
  <div class="disclaimer-banner__bar">
    <button
      class="disclaimer-banner__toggle"
      aria-label="Pokaż disclaimer"
      aria-expanded="true"
      aria-controls="disclaimer-banner__content"
    >
      <span class="disclaimer-banner__icon" aria-hidden="true">!</span>
      <span class="disclaimer-banner__chevron" aria-hidden="true">▾</span>
    </button>
    <div id="disclaimer-banner__content" class="disclaimer-banner__content">
      <!-- text injected from pl.disclaimer.full -->
    </div>
  </div>
</div>
```

**States:**

| State | Trigger | Visual | aria-expanded |
|-------|---------|--------|---------------|
| Expanded (default on first load) | — | Full text visible, amber tinted bar, chevron `▾` | `"true"` |
| Collapsed | User clicks chevron | Bar shrinks to 8px height with `!` button, chevron `▴` | `"false"` |

**Interaction contract:**

- Click chevron button → toggles expanded/collapsed
- Collapsed hover: `title` attribute on button shows full disclaimer text (progressive enhancement; Phase 5 will wire this to `@floating-ui/dom` tooltip)
- No dismiss. No close-forever. The component is permanently present in the DOM from first render.
- Keyboard: `Enter` / `Space` on button toggles state (native `<button>` behaviour)

**Persistence:**
- `localStorage` key: `pm300:disclaimer:collapsed:v1`
- Value: `"true"` = collapsed, `"false"` = expanded
- On load: read key; if missing, default to expanded
- On toggle: write key

**Position in layout:**
- `position: sticky; top: 0; z-index: 100;` — sits above `#ui-layer` (z-index 10) and above the Three.js canvas (z-index 1)
- Full viewport width
- `pointer-events: auto` (overrides the `#ui-layer` parent's `pointer-events: none`)

**Collapsed-state "stale visible" interpretation (from D-13):**
The `!` icon button is always visible in the viewport when collapsed. This satisfies UI-05 "widoczny stale" — permanent presence is met by the icon; full text expansion is user-triggered. A code comment above the component in `DisclaimerBanner.js` must document this interpretation explicitly to prevent future `dismiss=true` PRs.

## Integration Notes for Executor

1. **Position of `#disclaimer-banner` in HTML:** Insert as the first child of `<body>`, before `#three-canvas` and before `#ui-layer`. It must not be inside `#ui-layer` (which has `pointer-events: none`).

2. **CSS file:** All new disclaimer styles go into `style.css` (root). The `src/style.css` file is deleted in Phase Z hygiene — never edit it.

3. **JS module:** Implement as `src/DisclaimerBanner.js` — a plain ES module class `DisclaimerBanner` with `constructor()` (reads DOM, reads localStorage, sets initial state) and `mount()` (binds events). Imported in `src/main.js`, instantiated in `Application` constructor. Must be listed in `Application.dispose()` cleanup.

4. **i18n file:** `src/i18n/pl.js` exports a plain object `pl`. Structure:
   ```js
   export const pl = {
     disclaimer: { full: '...', short: '...', ariaLabel: '...', toggleExpand: '...', toggleCollapse: '...' },
     webgl: { contextLost: '...' },
     machineState: { oczekiwanieNaInspekcje: '...', gotowaDoPracy: '...', rozpedzanie: '...', wCyklu: '...', zatrzymana: '...', awaria: '...', trybWolny: '...' },
     errors: { 'E-OSLONA-NIEZAMKNIETA': '...', 'E-SPRZEGNIETO-PRZED-ROZPEDEM': '...' }
   };
   ```
   Error keys use the same string as the `errorCode` in scenario JSON (D-04). Executor must pre-populate all keys from the Copywriting Contract table above.

5. **No Three.js in `DisclaimerBanner.js`:** Boundary rule (INFRA-02). The banner is pure DOM.

6. **Code comment requirement (D-13):** The `DisclaimerBanner` class must have a JSDoc comment explaining the "widoczny stale" interpretation: collapsed state with `!` icon counts as "always visible" per D-13; `dismiss=true` must never be added.
